# 07. 潜在バグ台帳 — 未発火欠陥の管理(カタログ J カテゴリの実体)

「今までのデータではたまたま発生せず、正しく動作したように見えていた」欠陥の
全件台帳。各欠陥について **引き金条件 / 現行データが引き金を踏まない保証 /
MF での発火時挙動 / 移行後の典型挙動差 / 挑発データ** を1行で対応させる。
移行演習の採点では、検出・凍結/修正判断・挙動差再現の3点を本台帳と突合する。

## 運用ルール

1. **二重台帳の禁止**: ゴミデータ(dirty_ledger.csv=MF で正しく処理される汚れ)と
   本台帳(=MF でも欠陥、ただし未発火)は明確に区別する。前者は「移行で壊れる
   データ」、後者は「移行で挙動差が出る**コード**」。
2. **生成側の保証**: tools/ のデータ生成は本台帳の「保証」列を検証項目として持ち、
   通常データが引き金条件を1件も満たさないことを生成時にアサートする。
3. **バグ互換性テスト**: 挑発データ投入時の挙動(桁落ち値・読み飛ばし件数・
   アベンドコード)を MF 想定挙動として記録し、移行側の再現/差異を採点する。
4. 実装フェーズで欠陥を追加作込みした場合、**本台帳への登録なき潜在バグは
   混入禁止**(品質ゲートの管理外バグと区別できなくなるため)。
