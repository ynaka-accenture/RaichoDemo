# 08. コーディング規約 逆引き監査表 — 網羅性の体系的検証

作成方針の転換: 「思いつくままの帰納的積み上げ」から、**コーディング規約書・
研修テキストの章立てを演繹的に逆引き**する方式へ。典型的な現場規約
(構造化規約、命名規約、ファイル処理規約、演算規約、保守規約)の各ルールに
対し、「それを破るとどうなるか」が本カタログでカバーされているかを監査した。

判定: ◎=カバー済 / **NEW**=本監査で追加(→カタログ改訂) / 対象外=理由付き

## 監査結論

- 規約71項目の逆引きに対し、**12項目の漏れ**を2次にわたり検出。
- 漏れは「規格世代の混在」に集中しており、新カテゴリ **K. 規格・世代混在系**(6)
  と B-11 / D-10 / I-05、第2次監査で A-11 / B-12 / G-08 を追加し、
  **11カテゴリ85パターン**とした(docs/03 改訂)。
- 補足: RUTLDTC は既にこの観点の実例になっている — 全体は 74 様式
  (ピリオド終端・GO TO・EVALUATE 不使用)だが、参照修飾(B-09)は **85 規格の
  機能**であり、1本の中に規格の地層が存在する。
