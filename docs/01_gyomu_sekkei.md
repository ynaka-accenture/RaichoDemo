# 01. 業務設計 — 東西電力 営業料金システム RACS

## 1. 会社・システム設定(すべて架空)

- **東西電力株式会社**: 国内某地方の3県を供給区域とする架空の地域電力会社(実在の地域・事業者とは無関係)。需要家約6万件。
- システム名 **RACS**: 1985年初版稼働。汎用機(IBM z 系を想定)上の
  COBOL/CICS/VSAM を中核とし、2005年にスマートメーター実証(デマンド監視
  サブシステム: IMS/MQ/Db2)を増築。以後、料金制度改定・消費税改定・元号改定の
  保守を重ね、旧様式と新様式のコードが混在する典型的レガシー。
- 用語: 需要家(=顧客)、需給契約(=アカウント)、供給地点特定番号(22桁、
  契約のキー)、計器(=メーター)、検針、デマンド(30分平均電力)。

### 2.3 デマンド監視(オーソリ相当、オプション)

```
スマートメーター → MQ(RACS.DEMAND.REQ) → RAAUP00C(CICS MQトリガ起動)
   → RAAUP01C 判定サブ: IMS 需要家DB(JYUDBD)を SSA 照会
       ├ 契約電力と30分デマンドを突合(超過率判定)
       ├ 滞納停止フラグ照会(停止中なら遮断指令)
       └ 判定結果を MQ 応答 + Db2 警報履歴(TRACS.TALMHIST)に INSERT
バッチ: RBAUD01C(Db2履歴集計) / RBAUD02C(IMS DBロード)
```

判定コード: 00=正常 / 10=警報(契約電力の90%超) / 20=超過(100%超) /
30=遮断指令(滞納停止中または3コマ連続超過) — カードの「承認/限度額超過/
取引拒否」に対応。

## 3. 料金計算仕様(要点)

従量電灯B(主力メニュー)の月額 = 基本料金 + 電力量料金 + 燃料費調整額
+ 再エネ賦課金 − 口座振替割引、消費税は内税(歴史的経緯により旧外税レコード
が残存)。

- 基本料金: 契約アンペア × 単価(**円未満切捨て**)
- 電力量料金: 3段階(〜120kWh / 〜300kWh / 300kWh超)各段 **円未満四捨五入**
- 燃料費調整: 単価は**銭(小数2桁)で保持**し、使用量乗算後に**円未満切捨て**
- 再エネ賦課金: 円未満切捨て
- ※ 端数処理の方式が項目ごとに異なるのは実在制度に倣った意図的な複雑性。
  計算順序を変えると1円ズレる(罠 F-01)。

検針期間は「前回検針日翌日〜今回検針日」で、**使用月≠暦月**(20日締め境界、
罠 E-04)。計器交換時は検針値が逆転し、**マイナス差分を交換とみなす暗黙仕様**
(罠 F-04)。

## 4. 主要エンティティ

- 需要家(JYUMAST): 需要家番号(10桁)、漢字氏名(DBCS+SO/SI)、カナ氏名(半角カナ)、住所、外字含む
- 需給契約(KYKMAST): 供給地点特定番号(22桁 PIC X(22)=数字だが桁数超過のため文字格納)、契約種別、契約容量、単価改定世代
- 計器(MTRMAST): 計器製造番号、検定満期(和暦2桁)、乗率
- 検針実績(KENFILE): 指示数、使用量、検針日、検針員コード
- 料金・請求(RYOFILE)、収納(SHUFILE=RRDS 日別 + KSDS 明細)
- 単価表(TANKA 順編成 + プログラム内テーブルの二重管理: 罠 F-03)
